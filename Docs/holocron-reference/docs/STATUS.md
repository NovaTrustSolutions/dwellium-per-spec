## Architectural decisions (locked in — don't relitigate without new evidence)

1. **Build at Level 4 (Obsidian-style structured wiki + tags + agent-curated relationships), defer vector RAG (Level 5).** The Carpathia "magic" RAG is Level 4, not vector.
2. **PostgreSQL + tsvector for full-text search.** No pgvector in v1.
3. **Filesystem is the source of truth; the DB is a rebuildable index.** Nothing lives only in the DB without a disk artifact (wiki pages → `_Codex/Wiki/*.md`; syntheses will → `_Codex/Syntheses/*.md`).
4. **Wikilink-as-edge.** The ingestion agent inserts `[[Page Name]]` into documents; Holocron renders them as clickable links AND stores them as `rag_relationships` edges. Markdown convention, not vendor syntax.
5. **Task-based provider routing.** Gemini Flash for bulk transformation (tagging, wiki compile, triage); Claude Sonnet for synthesis + reports; Opus for deep planning. Never synthesis on Flash; never bulk on Sonnet.
6. **Boot-time self-healing.** Every structural invariant is checked + repaired idempotently on app start (`syncWorkspaceRoots`, `validateActiveConfigPaths`, `bootstrapMissingPages`, `deleteZombieWikiDocs`; `sweepOrphans` to be added). New invariants follow this pattern.
7. **The user is the quality gate for anything that mutates the knowledge base.** Nothing goes from an external source into the Codex without Review (the Foundry). Auto-ingest on file *create*; manual re-ingest for edits.
8. **Domaines *are* namespaces made visible.** No parallel system. Bridge namespaces (`__library__`, `__inbox__`) reach across Domaines; wiki rows do not (they're per-Domaine in the three-tier model). All Codex tab Domaine filters default to "(All Domaines)" and persist in their stores.
9. **iCloud Drive is the cloud transport, not Google Drive API.** OS handles sync; Holocron is the only app that ingests/indexes/rewrites.
10. **Local-first is non-negotiable.** Data stays local. Agents can be cloud.

## Trade-offs / decisions worth logging

- **Schema name normalization.** Entity in `entities.json` is `LaSonta
  Westbrook` (no middle initial), though the task brief referenced
  `LaSonta M. Westbrook`. I used the actual entity name for every id /
  assertion. No middle initial was introduced into the seed or test.
- **Visual-screenshot smoke step was set aside.** Documented in
  `Docs/Phase1/task_1_1_manual_smoke/README.md`. Two blockers combined:
  1. The runtime check `(import.meta as any)?.env?.VITE_USE_STATIC_API`
     uses optional chaining, which Vite's build-time replacer does not
     rewrite — so `VITE_USE_STATIC_API=true` set via CLI / `.env.local` /
     appended to `.env` was never picked up; browser consistently logged
     `[strataApi] mode=backend (/api/dwellium proxy)`.
  2. The sibling `ai-dashboard369-file-manager` backend is absent in this
     sandbox, so the normal e2e path (Playwright config auto-starts both
     servers) wasn't available. Route-mocking `/api/auth/me`, `/api/auth/
     refresh`, and `/api/dwellium/**` from Playwright got past login but
     the Strata dashboard rendered a minimal-permission view — the `god`
     role bypass is honored but some widget rendering looks gated on a
     richer permissions-shape than the mock returned.

  Rather than invest further time fighting the local-only env plumbing,
  I leaned on the automated contract test which exercises the exact data
  seam `OtherOccupantsSection` consumes. The strict CI gates cover the
  schema, seed, route handler, and test.
- **Smoke script kept out of the repo.** The exploratory Playwright
  script I wrote (`qualia-shell/scripts/task_1_1_smoke.mjs`) was deleted
  before commit — it depended on the non-robust env flag and route-mock
  permutation described above, so committing it would have been a hidden
  landmine. The `e2e/screenshot-baseline.spec.ts` suite is the correct
  long-term home for a Residents visual baseline; it should be paired
  with the eventual Linux Playwright baseline capture task.

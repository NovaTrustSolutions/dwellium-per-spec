- **`vi.useFakeTimers` + React 19 scheduler anti-pattern.** `vi.useFakeTimers()` mocks React 19's scheduler primitives (`MessageChannel` and microtask queue) that the React renderer uses for state-update commits in concurrent rendering mode. The `vi.useFakeTimers → vi.advanceTimersByTimeAsync → vi.useRealTimers` handoff is **empirically unsafe at jsdom + RTL + React 19 altitude** when the component-under-test has an async-cascade-to-render path: pending render commits stranded under fake primitives never fire on the real clock post-handoff. When the only Date-mocking need is `Date.now()` / `new Date()` anchoring (e.g., for "today" computation), use `vi.setSystemTime(...)` ALONE — it mocks the date without mocking scheduler primitives, leaving RTL's `waitFor()` to handle async settlement against the real clock deterministically.
